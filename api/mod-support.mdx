## Overview

Vista provides deep integrations with popular Minecraft mods, extending functionality and ensuring compatibility. All integrations are automatically enabled when the corresponding mod is detected.

## Exposure Integration

Vista integrates with the [Exposure](https://modrinth.com/mod/exposure) mod to support photograph and film items in picture frames.

### Picture Tape Item

Vista adds a special Picture Tape item that works with Exposure's photograph system:

```java ExposureCompat.java:33-37

    VistaMod.res("picture_tape"),
    () -> new PictureTapeItem(new Item.Properties()
        .component(PICTURE_TAPE_CONTENT.get(), new PictureTapeContent(List.of())))
);
```

### Picture Tape Content Component

The Picture Tape uses a custom data component to store photograph data:

```java ExposureCompat.java:26-31

    RegHelper.registerDataComponent(VistaMod.res("item_list"),
        () -> DataComponentType.<PictureTapeContent>builder()
            .persistent(PictureTapeContent.CODEC)
            .networkSynchronized(PictureTapeContent.STREAM_CODEC)
            .build());
```

### Picture Tape Menu

A dedicated menu allows players to view and manage picture tapes:

```java ExposureCompat.java:39-40

    RegHelper.registerMenuType(VistaMod.res("picture_tape"), PictureTapeMenu::fromBuffer);
```

<Note>

</Note>

### Example: Automated Camera Patrol

```lua
-- Wrap the view_finder peripheral
local camera = peripheral.wrap("view_finder")

-- Patrol pattern
local waypoints = {
    {yaw = 0,   pitch = 0,  zoom = 1},
    {yaw = 90,  pitch = 0,  zoom = 2},
    {yaw = 180, pitch = -30, zoom = 3},
    {yaw = 270, pitch = 0,  zoom = 1},
}

-- Patrol loop
while true do
    for _, point in ipairs(waypoints) do
        camera.setYaw(point.yaw)
        camera.setPitch(point.pitch)
        camera.setZoom(point.zoom)
        sleep(5)
    end
end
```

## Supplementaries Integration

Vista integrates with [Supplementaries](https://modrinth.com/mod/supplementaries) to add special shader effects and compatibility features.

### Custom Shader Support

Supplementaries' mob head shader system is integrated with Vista:

```java SuppCompat.java:20-25
public static ResourceLocation getShaderForItem(Item item) {
    if (item == VistaMod.TV_ITEM.get()) return null;
    if (item == QUARK_GLASS_PANE.get()) return RETRO_SHADER;
    String shaderForItem = MobHeadShadersManager.INSTANCE.getShaderForItem(item);
    return shaderForItem == null ? null : ResourceLocation.parse(shaderForItem);
}
```

```java JEICompat.java:22-25
    registration.registerSubtypeInterpreter(VistaMod.CASSETTE.get(), CassetteSubtypeInterpreter.INSTANCE);
```

#### Subtype Interpreter Implementation

```java JEICompat.java:27-40
public enum CassetteSubtypeInterpreter implements ISubtypeInterpreter<ItemStack> {
    INSTANCE;

    @Override
    public Object getSubtypeData(ItemStack stack, UidContext uidContext) {
        return stack.get(VistaMod.CASSETTE_TAPE_COMPONENT.get());
    }

    @Override
    public String getLegacyStringSubtypeInfo(ItemStack stack, UidContext uidContext) {
        Object component = stack.get(VistaMod.CASSETTE_TAPE_COMPONENT.get());
        return component == null ? "" : component.toString();
    }
}
```

<Note>
This ensures cassette tapes with different recordings are treated as distinct items in JEI's recipe and item lists.
</Note>

## Creating Custom Integrations

To add your own mod integration to Vista:

### Step 1: Add Detection Flag

Add your mod detection to `CompatHandler.java`:

```java
public static final boolean YOUR_MOD = PlatHelper.isModLoaded("your_mod_id");
```

### Step 2: Create Compatibility Class

Create a new class in `integration/your_mod/`:

```java
package net.mehvahdjukaar.vista.integration.your_mod;

public class YourModCompat {
    public static void init() {
        // Initialization code
    }
}
```

### Step 3: Register in CompatHandler

Add initialization to `CompatHandler.init()`:

```java
if (YOUR_MOD) YourModCompat.init();
```

### Step 4: Add Configuration (Optional)

If you need config options, implement:

```java
public static void addConfigs(ConfigBuilder builder) {
    // Add configuration options
}
```

And register in `CompatHandler.addConfigs()`:

```java
if (YOUR_MOD) YourModCompat.addConfigs(builder);
```

<Warning>
Always check the mod detection flag before accessing mod-specific classes to prevent ClassNotFoundException errors.
</Warning>

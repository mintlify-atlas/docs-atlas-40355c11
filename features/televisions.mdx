# Television Blocks

Television blocks are the core display devices in Vista. They can be connected together to form large multi-block screens and support cassette playback, live broadcasts, and special interactions.

## Multi-Block System

Televisions automatically connect to adjacent TV blocks to form larger displays. The mod uses a sophisticated grid detection system to create seamless multi-block structures.

### Connection Types

The `TVType` enum defines how each TV block connects to its neighbors:

```java
public enum TVType {
    SINGLE,      // Standalone TV
    CENTER,      // Connected on all sides
    TOP,         // Connected below, left, and right
    BOTTOM,      // Connected above, left, and right
    LEFT,        // Connected above, below, and right
    RIGHT,       // Connected above, below, and left
    TOP_LEFT,    // Connected below and right
    TOP_RIGHT,   // Connected below and left
    BOTTOM_LEFT, // Connected above and right (holds block entity)
    BOTTOM_RIGHT // Connected above and left
}
```

<Note>
Only `SINGLE` and `BOTTOM_LEFT` connection types have block entities. All other TVs in a multi-block structure reference the master block entity at the bottom-left position.
</Note>

### How Connection Works

When you place a TV block:

<Steps>
  <Step title="Neighbor Detection">
    The block checks all four adjacent positions (up, down, left, right) for other TV blocks with the same facing direction.
  </Step>
  
  <Step title="Grid Expansion">
    The `RectFinder` algorithm finds the maximum rectangular area that can be formed, up to the configured size limit.
  </Step>
  
  <Step title="State Assignment">
    Each TV in the grid gets assigned its appropriate `TVType` based on its position and connections.
  </Step>
  
  <Step title="Block Entity Transfer">
    If a cassette was in a TV that's no longer the bottom-left, it transfers to the new master block.
  </Step>
</Steps>

## Power States

TVs have three distinct power states that control their operation:

<AccordionGroup>
  <Accordion title="OFF - No Power">
    TV is completely off and displays nothing. No light is emitted and playback is stopped.
    
    ```java
    PowerState.OFF  // No redstone signal
    ```
  </Accordion>
  
  <Accordion title="DIRECT - Strong Power">
    TV receives direct redstone power from an adjacent source. This is the primary "on" state.
    
    ```java
    PowerState.DIRECT  // Adjacent redstone power
    ```
  </Accordion>
  
  <Accordion title="INDIRECT - Weak Power">
    TV receives indirect redstone power through blocks. Still considered "on" but marked as indirect.
    
    ```java
    PowerState.INDIRECT  // Indirect redstone signal
    ```
  </Accordion>
</AccordionGroup>

From `PowerState.java`:

```java
public boolean isOn() {
    return this != OFF;
}

public boolean isStrong() {
    return this == DIRECT;
}
```

<Warning>
When TVs are powered on, they emit light level 3. The power state affects all connected TVs in a multi-block structure.
</Warning>

## Redstone Integration

### Input - Power Control

TVs respond to redstone signals to control power state:

- **Direct Power**: Adjacent redstone sources (torches, repeaters, etc.)
- **Indirect Power**: Power through solid blocks
- **Signal Changes**: Power state updates propagate through the entire multi-block structure

```java
// From TVBlock.java:200-216
protected void neighborChanged(BlockState state, Level level, BlockPos pos, 
                               Block neighborBlock, BlockPos neighborPos, 
                               boolean movedByPiston) {
    if (neighborBlock == this) return;
    boolean powered = level.hasNeighborSignal(pos);
    PowerState oldPower = state.getValue(POWER_STATE);
    PowerState newPower = PowerState.direct(powered);
    
    if (newPower != oldPower)
        level.setBlock(pos, state.setValue(POWER_STATE, newPower),
                Block.UPDATE_KNOWN_SHAPE | Block.UPDATE_CLIENTS);
    
    if (oldPower.isOn() != newPower.isOn()) {
        // Update entire grid structure
        TVGridAccess gridAccess = new TVGridAccess(level, pos, state);
        Rect2D old = RectFinder.findMaxRect(gridAccess, Vec2i.ZERO, false);
        gridAccess.transform(old, old, null);
        gridAccess.applyChanges();
    }
}
```

### Output - Comparator

TVs provide analog output based on the inserted cassette:

```java
protected int getAnalogOutputSignal(BlockState state, Level level, BlockPos pos) {
    BlockEntity var5 = level.getBlockEntity(pos);
    if (var5 instanceof TVBlockEntity tv) {
        return tv.getComparatorOutput();
    }
    return 0;
}
```

## Cassette Playback

### Inserting and Ejecting

TVs use the `ItemDisplayTile` system from Moonlight Lib to manage cassette storage:

<Steps>
  <Step title="Insert Cassette">
    Right-click on any TV in a multi-block structure with a cassette item. The cassette transfers to the master block entity.
    
    ```java
    public SoundEvent getAddItemSound() {
        return VistaMod.CASSETTE_INSERT_SOUND.get();
    }
    ```
  </Step>
  
  <Step title="Auto-Play">
    When powered on, the TV begins playing the cassette content automatically.
  </Step>
  
  <Step title="Eject Cassette">
    Right-click again to eject. The cassette pops out as an item entity.
    
    ```java
    level.playSound(player, worldPosition, 
                   VistaMod.CASSETTE_EJECT_SOUND.get(),
                   SoundSource.BLOCKS, 1, 1);
    ```
  </Step>
</Steps>

### Playback Ticking

The TV tracks playback time and manages video/audio synchronization:

```java
public static void onTick(Level world, BlockPos pos, BlockState state, TVBlockEntity tv) {
    boolean powered = state.getValue(TVBlock.POWER_STATE).isOn();
    
    if (powered) {
        if (!tv.paused) tv.videoPlaybackTicks++;
    } else {
        tv.fadeAnimation.decrement();
        tv.videoPlaybackTicks = 0;
    }
    
    if (world.isClientSide && powered) {
        float duration = tv.videoSource.getVideoDuration();
        if (++tv.soundLoopTicks >= (duration)) {
            tv.soundLoopTicks = 0;
            SoundEvent sound = tv.videoSource.getVideoSound();
            if (sound != null) {
                world.playLocalSound(pos, sound, SoundSource.BLOCKS, 1, 1.0f, false);
            }
        }
    }
}
```

## Screen Rendering

The screen size is calculated based on the number of connected blocks:

```java
public int getScreenPixelWidth() {
    return Math.max(1, connectedTvsAmount) * 16 - EDGE_PIXEL_LEN;
}

public int getScreenPixelHeight() {
    return Math.max(1, connectedTvsAmount) * 16 - EDGE_PIXEL_LEN;
}
```

- **Edge Pixels**: 4 pixels reserved for the TV frame/bezel
- **Minimum Screen**: 12x12 pixels for a single TV (16 - 4)
- **Maximum Screen**: 124x124 pixels for an 8x8 grid

## Special Features

### Enderman Interaction

TVs have special behavior when an Enderman is visible through a broadcast:

```java
private TVEndermanObservationController observationController = null;

private void updateObservationController() {
    ItemStack displayedItem = this.getDisplayedItem();
    var uuid = displayedItem.get(VistaMod.LINKED_FEED_COMPONENT.get());
    this.observationController = uuid == null ? null : 
        new TVEndermanObservationController(uuid, this);
}
```

<Warning>
When viewing an Enderman through a linked broadcast, the Enderman will detect being watched and may become aggressive. This creates unique gameplay mechanics for security camera systems!
</Warning>

## Configuration

Key configuration option from `CommonConfigs`:

- **MAX_CONNECTED_TV_SIZE**: Maximum width/height of connected TV grids (default: 8)
  - Set to 1 to disable multi-block TVs entirely
  - Higher values allow larger screens but may impact performance

## Tips and Tricks

<AccordionGroup>
  <Accordion title="Building Large Displays">
    - Place TVs in a grid pattern, bottom-left to top-right
    - All TVs must face the same direction
    - Hold sneak while placing to prevent auto-connection
    - The cassette always goes in the bottom-left TV
  </Accordion>
  
  <Accordion title="Redstone Automation">
    - Use comparator output to detect when cassettes are inserted
    - Different cassettes can have different signal strengths
    - Hollow cassettes always output maximum signal (15)
    - Power the grid from any TV - the signal affects all connected blocks
  </Accordion>
  
  <Accordion title="Performance">
    - Larger TV grids update less frequently (staggered by position)
    - Enderman detection is throttled to every 27 ticks
    - Audio loops based on video duration to maintain sync
  </Accordion>
</AccordionGroup>

Vista is designed to work seamlessly with many popular Minecraft mods, providing enhanced functionality and compatibility. This guide covers all major mod integrations.

## Overview

Vista automatically detects and integrates with compatible mods at runtime. From `CompatHandler.java:14-19`, the mod checks for:

```java
public static final boolean EXPOSURE = PlatHelper.isModLoaded("exposure");
public static final boolean DISTANT_HORIZONS = PlatHelper.isModLoaded("distanthorizons");
public static final boolean COMPUTER_CRAFT = PlatHelper.isModLoaded("computercraft");
public static final boolean SUPPLEMENTARIES = PlatHelper.isModLoaded("supplementaries");
public static final boolean IRIS = PlatHelper.isModLoaded("iris") || PlatHelper.isModLoaded("oculus");
public static final boolean SODIUM = PlatHelper.isModLoaded("sodium") || PlatHelper.isModLoaded("embeddium");
```

No configuration is needed - integrations activate automatically when compatible mods are present.

## ComputerCraft Integration

<Note>
Requires **ComputerCraft: Tweaked** to be installed.
</Note>

Vista adds peripheral support for Viewfinders, allowing you to control cameras programmatically with ComputerCraft computers.

### Viewfinder Peripheral

When you connect a computer to a Viewfinder (using networking cables), it becomes a peripheral with the type `"view_finder"`.

#### Available Methods

From `ViewFinderPeripheral.java:21-53`, the following Lua functions are available:

<AccordionGroup>
  <Accordion title="setYaw(value)" icon="code">
    Sets the horizontal rotation of the camera.
    
    **Parameters:**
    - `value` (number): Yaw angle in degrees
    
    **Example:**
    ```lua
    local viewfinder = peripheral.find("view_finder")
    viewfinder.setYaw(90)  -- Point camera east
    ```
  </Accordion>

  <Accordion title="getYaw()" icon="code">
    Returns the current horizontal rotation.
    
    **Returns:** `number` - Current yaw in degrees
    
    **Example:**
    ```lua
    local yaw = viewfinder.getYaw()
    print("Camera facing: " .. yaw .. " degrees")
    ```
  </Accordion>

  <Accordion title="setPitch(value)" icon="code">
    Sets the vertical tilt of the camera.
    
    **Parameters:**
    - `value` (number): Pitch angle in degrees (-90 to 90)
    
    **Example:**
    ```lua
    viewfinder.setPitch(-45)  -- Tilt camera up 45 degrees
    ```
  </Accordion>

  <Accordion title="getPitch()" icon="code">
    Returns the current vertical tilt.
    
    **Returns:** `number` - Current pitch in degrees
    
    **Example:**
    ```lua
    local pitch = viewfinder.getPitch()
    ```
  </Accordion>

  <Accordion title="setZoom(zoom)" icon="code">
    Sets the zoom level of the camera.
    
    **Parameters:**
    - `zoom` (integer): Zoom level from 1 to 44
    
    **Example:**
    ```lua
    viewfinder.setZoom(20)  -- Set to 20x zoom
    ```
    
    <Note>Zoom is automatically clamped between 1 and 44.</Note>
  </Accordion>

  <Accordion title="getZoom()" icon="code">
    Returns the current zoom level.
    
    **Returns:** `integer` - Current zoom level (1-44)
    
    **Example:**
    ```lua
    local zoom = viewfinder.getZoom()
    ```
  </Accordion>
</AccordionGroup>

#### Example: Sweeping Security Camera

```lua
-- Sweep camera back and forth
local viewfinder = peripheral.find("view_finder")
local minYaw = -45
local maxYaw = 45
local speed = 2

while true do
    -- Sweep right
    for yaw = minYaw, maxYaw, speed do
        viewfinder.setYaw(yaw)
        sleep(0.1)
    end
    
    -- Sweep left
    for yaw = maxYaw, minYaw, -speed do
        viewfinder.setYaw(yaw)
        sleep(0.1)
    end
end
```

## Supplementaries Integration

<Note>
Requires **Supplementaries** to be installed.
</Note>

Supplementaries adds several enhancements to Vista's functionality.

### Wrench Compatibility

Viewfinders can be rotated using Supplementaries wrenches:

- **Manual Rotation**: Right-click with a wrench to rotate the Viewfinder block
- **Axis-Specific**: Rotation respects the axis you're targeting
- **Angle Preservation**: The camera's pitch/yaw adjusts automatically to maintain the same view direction

From `ViewFinderBlock.java:185-205`, the rotation system:

```java
public Optional<BlockState> getRotatedState(BlockState state, LevelAccessor levelAccessor, 
                                            BlockPos blockPos, Rotation rotation, 
                                            Direction axis, @Nullable Vec3 hit) {
    boolean ccw = rotation == Rotation.COUNTERCLOCKWISE_90;
    return BlockUtil.getRotatedDirectionalBlock(state, axis, ccw)
           .or(() -> Optional.of(state));
}
```

This allows for:

## Iris Shaders Integration

<Note>
Requires **Iris Shaders** or **Oculus** (Forge version) to be installed.
</Note>

Vista includes special compatibility for shader packs to ensure Viewfinder cameras render correctly.

### Shader Compatibility Mode

From `IrisCompat.java:132-136`, Vista provides a config option:

```toml
# Attempts to disable iris shaders in the live feed view
iris_off_hack = true
```

When enabled (default), Vista:
- Temporarily disables shaders during Viewfinder rendering
- Prevents shader artifacts in camera feeds
- Restores shader state after rendering
- Maintains shadow and lighting information

### Technical Details

The integration uses a sophisticated state preservation system (`IrisCompat.java:32-53`):

```java
public static Runnable decorateRendererWithoutShaderPacks(Runnable renderTask) {
    return () -> {
        boolean oldShadowActive = ShadowRenderer.ACTIVE;
        OldRenderState oldState = OldRenderState.loadFrom(CapturedRenderingState.INSTANCE);
        WorldRenderingPipeline oldPipeline = getCurrentPipeline(lr);
        
        try {
            ShadowRenderer.ACTIVE = false;
            VISTA_RENDERING.set(true);
            renderTask.run();
        } finally {
            // Restore all state
        }
    };
}
```

This ensures:
- No conflicts with shader rendering pipelines
- Proper fog and lighting
- Compatible with bobbing effects
- Works with custom shader packs

<Tip>
If you experience visual glitches with specific shader packs, try toggling the `iris_off_hack` config option.
</Tip>

### Subtype Interpretation

The integration uses component data to distinguish cassettes:

```java
public Object getSubtypeData(ItemStack stack, UidContext uidContext) {
    return stack.get(VistaMod.CASSETTE_TAPE_COMPONENT.get());
}
```

This means cassettes with different video content are treated as separate items in JEI, even though they're the same item type.

## Troubleshooting

<AccordionGroup>
  <Accordion title="ComputerCraft Peripheral Not Detected" icon="computer">
    Ensure:
    - ComputerCraft: Tweaked is installed and loaded
    - The Viewfinder is properly connected via networking cable
    - You're using `peripheral.find("view_finder")` with the correct type name
  </Accordion>

  <Accordion title="Shader Effects Not Working" icon="wand-magic-sparkles">
    Check that:
    - Supplementaries is installed
    - You're using compatible items (stained glass panes, supported mob heads)
    - The Viewfinder isn't locked (items can't be inserted when locked)
  </Accordion>

  <Accordion title="Picture Tapes Not Available" icon="image">
    Verify:
    - Exposure mod is installed
    - Both mods are compatible versions
    - The Picture Tape item appears in the creative inventory under Tools & Utilities
  </Accordion>

  <Accordion title="Shader Glitches with Iris" icon="eye">
    Try:
    - Toggling the `iris_off_hack` config option
    - Updating to the latest Iris/Oculus version
    - Testing with different shader packs
    - Reporting specific issues to the Vista issue tracker
  </Accordion>
</AccordionGroup>
